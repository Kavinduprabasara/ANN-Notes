01 Artificial Intelligence — Foundations
What is Intelligence?
  • Intelligence is not absolute — it is comparative and always depends on context. A beggar may be more intelligent than a professor in the context of begging (expert in that environment).
  • Humans are not universally intelligent — intelligence depends on domain knowledge; people have different expertises.
  • Can an LLM understand what we are saying? No — it processes patterns without genuine comprehension.
  • There are many ways to define AI, so any perspective should not be discouraged. We need algorithms to simulate intelligent concepts drawing from maths, biology, and neuroscience.
AI Definition & Implementation

AI is the science and engineering of making intelligent machines, especially intelligent computer programs / intelligent agents. It is related to the similar task of using computers to understand human intelligence.

① Perceive & Understand
Ability to perceive and understand the world based on available knowledge models. Percieveness is not isolated — it is connected with the knowledge model.
② Inferencing on Knowledge
Perform reasoning on accumulated knowledge to achieve objectives and goals. Produce decisions. Includes Learning & Adaptation.
③ Planning & Execution
Generate a plan of actions to execute decisions on the environment. Once reasoning is done, execution is easier.
💡 Key Insight: Percieveness depends on context — it shows what we want to see. It is defined by our desires and our knowledge model. The same object (e.g., a pen) is perceived differently by a doctor, a child, or an artist.
Key AI Techniques
Machine Learning
Techniques that create knowledge models on machines by using structured and unstructured datasets. NLP & Computer Vision are based on percieveness.
Multiagent Systems
Modelling complex systems whose behaviour is intrinsically difficult to model due to dependencies — uses large numbers of autonomous agents.
Evolutionary Computing
Based on Darwin's principles of natural selection. Mainly used for heuristic optimisation.
Natural Language Processing
Gives machines the ability to read, understand and derive meaning from human languages. Speech processing: listen, percieve, then talk.
Fuzzy Logic
Also known as multi-valued knowledge. Handles uncertainty from imprecision and imperfect knowledge.
Cognitive / Expert Systems
Knowledge representation in terms of symbolic logics. Simulates the judgement of human decision processes.
02 Knowledge Representation
Two Types of Knowledge Representations
1. Rules & Facts
IF-THEN rule sets. Example: If there is a vehicle on the left → check your right. Fuzzy logic and expert systems are rule-based knowledge models.
2. Data-Oriented Models
Knowledge represented as tabular data, mathematical functions, graphs/images, or probability distributions. No fixed format — we can develop our own knowledge base.
📌 Having a proper knowledge model allows us to do better reasoning. If we don't have a proper knowledge model, understanding is also weak. Reasoning is stronger when the knowledge model is very strong; reasoning is weak when the knowledge model is weak.

Accuracy vs Speed Trade-off in Knowledge
  • When developing a knowledge model, consider: how fast to retrieve, how fast to store, and accuracy.
  • Short instructions are easy to develop but accuracy is low (it indirectly says what should be there).
  • Long, detailed instructions take longer to retrieve, but are more accurate.
03 Learning — What It Is & How It Works
Definitions
  • Learning = developing a process that develops a knowledge model.
  • Data = nothing but experiences. When you get an experience, you apply your existing knowledge model, get an output, receive feedback, and update the knowledge model.
  • The more experience in an environment, the more confident you become.
  • Getting the input, updating the knowledge model from experiences, and improving the model is called the learning process.

04 Data Preparation
Data Coding (Encoding)

Variables can have numerical or text values. When feeding text values to a neural network, everything must be expressed in 1s and 0s. Converting text values into numerical values is called encoding.

One-Hot Encoding
Creates binary columns for each category. No ordinal relationship implied. E.g., [cat, dog, bird] → [1,0,0], [0,1,0], [0,0,1]. Best for nominal categorical data.
Integer (Label) Encoding
Assigns integer values 1, 2, 3… to categories. Straightforward; does not expand feature space. Works well with tree-based algorithms. Problem: implies order that may not exist.
Ordered Label Encoding
Use when a monotonic relationship exists — e.g., preference: none < low < mid < high. Integer encoding is suitable here because order matters.
⚠️ When applying codings, choose carefully. For colour (no natural order), integer encoding is NOT suitable — it falsely implies 3 > 2 > 1. Use one-hot instead.
Feature Scaling

Feature magnitude matters because: the scale of a variable directly influences regression coefficients, variables with larger magnitudes dominate, and gradient descent converges faster when features are on similar scales.

Standardization (Z-score)
x̄ = (x − mean(x)) / std(x)
Range: approximately (−3, +3)
Centres data around 0 with unit variance. Robust for most algorithms.
Min-Max Normalization
x̄ = (x − min(x)) / (max(x) − min(x))
Range: [0, 1]
Scales to fixed range. Very sensitive to outliers.
  • Euclidean distances are sensitive to feature magnitude → always scale before KNN, SVM.
  • Feature scaling helps decrease the time to find support vectors for SVMs.
  • Gradient descent converges faster when features are on similar scales.
Custom Discretization

Discretization of continuous variables like Age — divide [0–10] as "kids", [10–25] as "teenagers", etc. Example using pandas:

// Python — pd.cut() discretization
bins = [0, 10, 25, 65, 250]
labels = ['0-10', '10-25', '25-65', '>65']
df['Age'] = pd.cut(df.Age, bins=bins,
                 labels=labels,
                 include_lowest=True)
Class Imbalance Handling
Undersampling
Tomek Links: Remove majority class samples that are closest to minority class. Near Miss: Select majority samples nearest to minority examples. Edited Nearest Neighbours: Drop majority points not in your boundary.
Evaluation Metrics
• Accuracy (Balanced) — balanced datasets
• AUC-ROC — slightly imbalanced
• Precision & Recall / F1 — significantly imbalanced
05 Error Functions & Loss
Regression Error
// Sum of Squared Errors (SSE)
e = real value − predicted value
E = Σ e²

Use error E to update θ (parameters).
ΔΘ = update step proportional to E

In regression: xi → KM(Θ) → ŷ compared against real y. Error drives parameter update.

Classification Error
// Delta / Indicator Error (Classification)
e = δ(ŷ − y)

If ŷ and y are the same → δ(ŷ, y) = 0
If ŷ and y are different → δ(ŷ, y) ≠ 0

Decision boundary: y = θ̂₁x₁ + θ̂₂x₂ + ĉ. Anything above the line is one class, below is another.

Confusion Matrix & Key Metrics
// Confusion Matrix
// Key Classification Metrics
Accuracy = (TP + TN) / (TP + TN + FP + FN)
Precision = TP / (TP + FP)
Recall = TP / (TP + FN)
F1 Score = 2 × (Precision × Recall) / (Precision + Recall)

FN ∝ 1/FP (FN is inverse proportional to FP)
In reality, FN never will be zero.
Goal: minimise both FP & FN.
Perceptron Learning Rule
// Perceptron Weight Update Rule
w(t+1) = w(t) + η · [y(t) − ŷ(t)] · x(t)

where:
η = learning rate
y = true label
ŷ = predicted output (using sgn function)
x = input vector

z = wx + b
y = sgn(z) where sgn(z) = +1 if z ≥ 0, else −1
06 The Neuron Model
Biological Inspiration

The artificial neuron is inspired by biological neurons. Inter-neuron connection strengths (weights) are used to store acquired knowledge. Inputs are received, weighted, summed, and passed through an activation function.

Single Neuron Architecture
// Single artificial neuron (with bias)
// Neuron computation
u = x₁w₁ + x₂w₂ + ... + xₙwₙ (weighted sum)
V = u + b (with bias)
V = w·xᵀ + b (matrix form with bias)
V = wx (without bias term)

y = f(u) (output = activation function applied to weighted sum)
// Weight matrix initialization
W^input → shape: (d+1) × #receiving_neurons
W^output → shape: 1 × #neurons

w¹ = [w₀ w₁ w₂ w₃] (row vector, 1×(d+1))
b¹ = [w₀]₁ₓ₁
xᵀ = [x₁ x₂ x₃] (1×d row)
Parallel vs Sequential Processing
  • Within a layer (L₁): neurons process information in parallel — we can calculate y₂, y₃, y₁ simultaneously since we know inputs x₁ to xₙ.
  • Across layers: processing is sequential — layer 1 output feeds layer 2.
  • There is no restriction to connect neurons parallelly (parallel processing).
07 Activation Functions
Why Activation Functions?

Activation functions introduce non-linearity into the network. Without them, the entire network collapses to a single linear transformation no matter how many layers exist. Hidden layers bring "non-linearity" to the network.

Core Activation Functions
Sigmoid (Logistic)
σ(x) = 1 / (1 + e⁻ˣ)
Range: (0, 1) · S-shaped curve
Used for binary classification output. Smooth, differentiable.
Vanishing gradient in deep nets — gradients become very small.
Tanh (Hyperbolic Tangent)
tanh(x) = (eˣ − e⁻ˣ) / (eˣ + e⁻ˣ)
Range: (−1, +1) · Zero-centred
Used in hidden layers when zero-centred output needed. Stronger gradients than sigmoid near zero.
Vanishing gradient still present in very deep nets.
ReLU (Rectified Linear Unit)
f(x) = max(0, x)
Range: [0, +∞)
Default for hidden layers in modern networks. Computationally cheap; avoids vanishing gradient.
Dying ReLU — neurons can get stuck outputting 0 forever.
Leaky ReLU
f(x) = max(αx, x) (α ≈ 0.01)
Range: (−∞, +∞)
Fixes Dying ReLU by allowing small negative slope. Enables backpropagation even for negative inputs.
Softmax
σ(xᵢ) = eˣⁱ / Σⱼ eˣʲ
Range: (0,1), sums to 1
Used at the output layer for multi-class classification. Converts raw scores to probability distribution. Outputs are interrelated (not independent like sigmoid).
Signum (Sign Function)
sgn(z) = +1 if z≥0, else −1
Range: {−1, +1}
Used in perceptron (binary hard threshold). Not differentiable at 0; not usable with backprop.

Vanishing Gradient Problem
⚠️ In deep networks with Sigmoid or Tanh, gradients can become extremely small during backpropagation, making it difficult for early layers to learn. This is why ReLU is preferred for hidden layers in modern deep networks.

Strongly vs Weakly Connected
  • Strongly connected: the weight value is large, but input might be zero → no actual activation.
  • Weakly connected: input is one but weight is almost zero.
  • Adjusting weights properly allows the network to identify different patterns in the dataset.
💡 If you adjust the weights of a neural network properly, you should be able to identify different patterns in the dataset. The network distinguishes patterns by activation of weight values and gives you different outputs.
Sequence Modelling (RNN / LSTM)
  • Language modelling uses feedforward networks where all features are given as one vector, or sequence models (RNNs, LSTMs).
  • LSTMs are sequence modelling networks with feedback — they carry context across time steps.
  • Example: predicting the next word in "I am Subha Fernando" — feed "I", predict "am"; feed "am", predict "Subha"; etc.
09 Training the Network
Backpropagation

Backpropagation (Backward Propagation of Errors) is the core algorithm for training ANNs. It computes the gradient of the loss function with respect to every weight using the chain rule of calculus, then updates weights via gradient descent.

// Weight update rule (gradient descent)
w_new = w_old − η · (∂Loss / ∂w_old)
b_new = b_old − η · (∂Loss / ∂b_old)

w(t+1) = w(t) + Δw
Δw = −η · ∂E/∂w (for regression: E = Σe²)

Chain rule (single layer):
∂E/∂wᵢ = (∂E/∂o) · (∂o/∂z) · (∂z/∂w)
η = learning rate (hyperparameter controlling step size)
    1. Forward pass: compute output ŷ layer-by-layer using current weights.
    2. Compute loss: E = Σ(y − ŷ)² (regression) or cross-entropy (classification).
    3. Backward pass: apply chain rule from output layer back to input — compute ∂E/∂w for every weight.
    4. Update weights: w ← w − η·∂E/∂w for all parameters.
Gradient Descent Variants
Method Batch Size Update Frequency Notes
Full Batch GD All 1000 samples Once per epoch Stable but slow; memory-intensive
Mini-Batch GD 200 samples (e.g.) 5× per epoch (1000÷200) Best trade-off; standard practice
Online (Stochastic) 1 sample After every sample Noisy but fast; update Δw immediately
✅ Epochs: One epoch = the full training dataset passed through the model once. If 5 epochs → iterate the full sample 5 times. Shuffling the dataset between epochs prevents the network from learning order dependencies.
Early Stopping & Validation
// Training vs Validation error across epochs
  • After each epoch, call the validation dataset (no weight updates — just predict).
  • If validation error starts increasing while training error decreases → overfitting begins.
  • Early Stopping: stop training at the epoch where validation error is lowest.
  • At early stop: save weights (ω_sw), test on test set → final evaluation.
// Training with validation loop
Epoch 1: train on mini-batch → update ω̂
validate with val data → compute val_err (no update)
Epoch 2: ω̂ = ω̂_old + Δω
re-apply val data → compare val_err
...
Early stop when val_err increases
→ final test: ω_sw → ω̂_st → E_test
Input Shuffling
  • After the first input x₁, adjust ω. Then comes x₂ — no dependency from x₁ to x₂ (fresh network).
  • We can shuffle training samples because inputs are independent of each other.
  • Shuffling during training breaks any independence/order between data points.
10 Deep Learning Patterns & Summary
Deep Learning Types: Overview
Deep Learning
Multi-layer neural networks. Hidden layers learn increasingly abstract representations. Non-linearity introduced by activation functions in every hidden layer.
Fuzzy Learning
Combines fuzzy logic with neural learning to handle imprecise/uncertain knowledge. Multi-valued reasoning.
Statistical Learning
Probability-based models. Bayesian learning, Gaussian processes. Knowledge represented as probability distributions.
Full Pipeline Summary
  1. 1
    Define Problem — regression (continuous output) or classification (discrete labels).
  2. 2
    Prepare Data — encode categorical variables, scale features (standardize / min-max), handle class imbalance, train/test split.
  3. 3
    Initialize Weights — random small values (breaks symmetry so neurons learn differently).
  4. 4
    Forward Pass — compute V = WX + b, then apply activation f(V) at each layer.
  5. 5
    Compute Loss — E = Σe² (regression) or cross-entropy (classification).
  6. 6
    Backpropagation — chain rule to compute ∂E/∂w for all weights, layer by layer (output → input).
  7. 7
    Update Weights — w ← w − η·∂E/∂w. Repeat for each mini-batch / sample (stochastic).
  8. 8
    Validate & Early Stop — monitor validation error per epoch; stop when val_err begins rising.
  9. 9
    Evaluate on Test Set — use saved weights (ω_sw) to compute final test metrics: accuracy, AUC-ROC, precision/recall.

Common Pitfalls & Tips
⚠ Vanishing Gradient
Occurs with Sigmoid/Tanh in deep nets. Fix: use ReLU in hidden layers.
⚠ Dying ReLU
Neurons permanently outputting 0. Fix: Leaky ReLU, careful initialization.
⚠ Overfitting
Model memorises training data. Fix: early stopping, dropout, regularization, more data.
💡 Learning Rate
Too large → overshoots minimum. Too small → very slow convergence. Use adaptive methods (Adam, RMSprop) for best results.
💡 Weight Init
Always initialize randomly (not all zeros) to break symmetry — otherwise all neurons in a layer learn the same thing.
💡 Shuffle Data
Shuffle dataset between epochs. Breaks order-dependencies and helps the network generalize.
